# CI Pipeline Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a GitHub Actions CI pipeline that typechecks all three workspaces, runs the vitest suites (node + client/jsdom), and runs the Playwright e2e suite on every push to `main` and every PR — with npm and Playwright-browser caching, superseded-run cancellation, a failure-report artifact, and a status badge in the README.

**Architecture:** One workflow file `.github/workflows/ci.yml` with two independent jobs: `test` (typecheck + unit tests) and `e2e` (full build + Playwright). The repo is an npm-workspaces monorepo (`client`/`server`/`shared`); `shared` is a composite TS project that `client` and `server` reference, so `shared` must be built before anything else typechecks or tests. Playwright's config already boots the production server itself (`webServer` → `NODE_ENV=production npm start`, health-checked at `/api/health`) and is CI-aware (`workers: 1`, `retries: 2`, `forbidOnly`), so the e2e job only needs to build and run `playwright test` — no manual server start. Only the `chromium` project is configured, so we install only that browser.

**Tech Stack:** GitHub Actions (`actions/checkout@v4`, `actions/setup-node@v4`, `actions/cache@v4`, `actions/upload-artifact@v4`), Node 20 (matches `Dockerfile` `node:20-alpine`; there is no `.nvmrc` and no `engines` field), npm workspaces, TypeScript 5.4, vitest 4, Playwright 1.58.

**Repo facts this plan relies on (verified 2026-07-07):**

- No `.github/` directory exists — nothing to migrate.
- Remote is `github.com:splaxter/kritis-sim`; default branch is `main`.
- Root scripts: `test` = `npm run build -w shared && vitest run` (node env, excludes `**/*.browser.test.tsx`); `test:client` = `npm run build -w shared && vitest run --root client --config vitest.config.ts` (jsdom browser tests); `test:e2e` = `npm run build && playwright test`. CI mirrors these but builds `shared` once instead of three times.
- `shared/tsconfig.json` sets `composite: true` → `tsc --noEmit -p shared` is **invalid** ("Composite projects may not disable emit"). Building `shared` *is* its typecheck. `npx tsc --noEmit -p client/tsconfig.json` and `-p server/tsconfig.json` both exit 0 today (verified locally) once `shared/dist` exists.
- `playwright.config.ts`: single `chromium` project, `reporter: 'list'` → **no `playwright-report/` directory is produced today**. Task 1 fixes that for CI so the failure artifact has content. Traces land in `test-results/` (`trace: 'on-first-retry'`).
- There is no lint script anywhere (no eslint config in any workspace) — CI's "lint" gate is `tsc --noEmit`. Do not invent an eslint setup (YAGNI).

## Task 2: Create the CI workflow

**Files:**
- Create: `/Users/timoklinge/Projekte/kritis_game/.github/workflows/ci.yml`

**Steps:**

1. Create the directory and file:

   ```bash
   mkdir -p /Users/timoklinge/Projekte/kritis_game/.github/workflows
   ```

2. Write `.github/workflows/ci.yml` with exactly this content:

   ```yaml
   name: CI

   on:
     push:
       branches: [main]
     pull_request:

   concurrency:
     group: ci-${{ github.workflow }}-${{ github.ref }}
     cancel-in-progress: true

   jobs:
     test:
       name: Typecheck & unit tests
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4

         - uses: actions/setup-node@v4
           with:
             node-version: 20
             cache: npm

         - name: Install dependencies
           run: npm ci

         # shared is a composite project referenced by client and server;
         # building it is also its typecheck (composite forbids --noEmit).
         - name: Build shared workspace
           run: npm run build -w shared

         - name: Typecheck client
           run: npx tsc --noEmit -p client/tsconfig.json

         - name: Typecheck server
           run: npx tsc --noEmit -p server/tsconfig.json

         # Mirrors `npm test` minus the redundant shared rebuild.
         - name: Unit tests (node)
           run: npx vitest run

         # Mirrors `npm run test:client` — jsdom *.browser.test.tsx suite.
         - name: Unit tests (client/jsdom)
           run: npx vitest run --root client --config vitest.config.ts

     e2e:
       name: Playwright e2e
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4

         - uses: actions/setup-node@v4
           with:
             node-version: 20
             cache: npm

         - name: Install dependencies
           run: npm ci

         - name: Cache Playwright browsers
           id: playwright-cache
           uses: actions/cache@v4
           with:
             path: ~/.cache/ms-playwright
             key: playwright-${{ runner.os }}-${{ hashFiles('package-lock.json') }}

         - name: Install Playwright chromium
           if: steps.playwright-cache.outputs.cache-hit != 'true'
           run: npx playwright install --with-deps chromium

         - name: Install Playwright OS deps (cache hit)
           if: steps.playwright-cache.outputs.cache-hit == 'true'
           run: npx playwright install-deps chromium

         - name: Build all workspaces
           run: npm run build

         # playwright.config.ts starts the production server itself
         # (webServer: NODE_ENV=production npm start, health check on
         # /api/health) and is CI-aware (workers: 1, retries: 2).
         - name: Run e2e tests
           run: npx playwright test

         - name: Upload Playwright report
           if: failure()
           uses: actions/upload-artifact@v4
           with:
             name: playwright-report
             path: |
               playwright-report/
               test-results/
             retention-days: 7
   ```

3. Validate the YAML parses:

   ```bash
   cd /Users/timoklinge/Projekte/kritis_game && node -e "const {readFileSync}=require('fs');const yaml=readFileSync('.github/workflows/ci.yml','utf8');console.log(yaml.split('\n').length,'lines, non-empty:',yaml.length>0)"
   npx --yes yaml-lint .github/workflows/ci.yml 2>/dev/null || python3 -c "import yaml,sys;yaml.safe_load(open('.github/workflows/ci.yml'));print('YAML OK')"
   ```

   Expected output: `YAML OK` (or yaml-lint's success message). If python3/pyyaml is unavailable, skip — step 4's push is the authoritative validation.

4. Commit:

   ```bash
   git add .github/workflows/ci.yml
   git commit -m "ci: add GitHub Actions pipeline (typecheck, unit, e2e)"
   ```

## Task 4: Verify the pipeline on GitHub

**Files:** none (verification only)

**Steps:**

1. Push the current branch (this plan lives on `feat/blackout-slice`; any pushed branch with an open PR — or a direct push once merged to `main` — triggers the workflow. If no PR exists yet, opening one is what triggers the `pull_request` run):

   ```bash
   git push -u origin HEAD
   ```

   Expected output: branch pushed; if a PR is open for the branch, a `CI` run starts immediately.

2. Confirm GitHub registered the workflow:

   ```bash
   gh workflow view CI
   ```

   Expected output: workflow name `CI`, state `active`, file `.github/workflows/ci.yml`, plus recent runs (may be empty until the first trigger).

3. Watch the run to completion:

   ```bash
   gh run watch --exit-status
   ```

   (If multiple runs exist, pick the run id from `gh run list --workflow=CI --limit 3` and pass it: `gh run watch <run-id> --exit-status`.)

   Expected output: both jobs `Typecheck & unit tests` and `Playwright e2e` end with a green check; command exits 0. On failure it exits non-zero — inspect with `gh run view <run-id> --log-failed`, and for e2e failures download the artifact: `gh run download <run-id> -n playwright-report`.

4. Verify concurrency cancellation works (optional smoke check): push two commits in quick succession to the same PR branch and run `gh run list --workflow=CI --limit 3` — the older run should show status `cancelled`.

5. Check the badge renders: open `https://github.com/splaxter/kritis-sim/blob/<branch>/README.md` — badge shows `CI passing` once a `main` run has completed (it reads `?branch=main`, so it stays "no status" until the workflow first runs on `main` after merge).
